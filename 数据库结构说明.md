# 数据库结构说明（MySQL）

> 基于 `server/db.ts` 中 `initDb()` 的实际建表语句整理。

## 1. 数据库概览

- 数据库名：`jiyun`（默认，来自 `DB_NAME`）
- 字符集：`utf8mb4`
- 存储引擎：`InnoDB`
- 主体业务表：
  - 用户：`users`
  - 包裹：`parcels`
  - 订单：`orders`
  - 短信验证码：`otp_codes`
  - 管理员：`admin_users`
  - 管理审计日志：`admin_audit_logs`

> 说明：当前代码未定义数据库层外键约束（仅通过业务字段关联）。

---

## 2. 表结构与字段说明

## 2.1 `users`（用户表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 用户主键ID |
| username | VARCHAR(64) | UNIQUE, NOT NULL | 登录用户名（系统内唯一） |
| password | VARCHAR(255) | NOT NULL | 登录密码哈希值 |
| phone | VARCHAR(32) | UNIQUE | 手机号（可空，唯一） |
| email | VARCHAR(255) | 无 | 邮箱（可空） |
| real_name | VARCHAR(255) | 无 | 真实姓名（可空） |
| address | VARCHAR(255) | 无 | 地址（可空） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

## 2.2 `otp_codes`（短信验证码表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 验证码记录ID |
| phone | VARCHAR(32) | NOT NULL, INDEX `idx_otp_phone` | 目标手机号 |
| code | VARCHAR(16) | NOT NULL | 验证码 |
| expires_at | DATETIME | NOT NULL, INDEX `idx_otp_expires` | 过期时间 |
| verified | TINYINT(1) | DEFAULT 0 | 是否已验证（0=未验证，1=已验证） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

---

## 2.3 `admin_users`（后台管理员表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 管理员ID |
| username | VARCHAR(64) | UNIQUE, NOT NULL, INDEX `idx_admin_username` | 管理员登录名 |
| password | VARCHAR(255) | NOT NULL | 管理员密码哈希 |
| email | VARCHAR(255) | NOT NULL | 管理员邮箱 |
| role | VARCHAR(32) | DEFAULT 'admin' | 管理员角色 |
| status | VARCHAR(32) | DEFAULT 'active' | 管理员状态 |
| last_login | DATETIME | 无 | 最后登录时间（可空） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 角色与状态（接口层约束）

- `role` 合法值：`admin`、`super_admin`
- `status` 合法值：`active`、`disabled`

---

## 2.4 `parcels`（包裹表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 包裹ID |
| user_id | INT | NOT NULL, INDEX `idx_parcels_user` | 所属用户ID（逻辑关联 `users.id`） |
| tracking_number | VARCHAR(128) | UNIQUE, NOT NULL, INDEX `idx_parcels_tracking` | 包裹单号（唯一） |
| origin | VARCHAR(255) | NOT NULL | 来源 |
| destination | VARCHAR(255) | NOT NULL | 目的地 |
| weight | DOUBLE | 无 | 重量（可空） |
| length_cm | DOUBLE | 无 | 长度（可空，单位 cm） |
| width_cm | DOUBLE | 无 | 宽度（可空，单位 cm） |
| height_cm | DOUBLE | 无 | 高度（可空，单位 cm） |
| volume | DOUBLE | 无 | 体积（可空） |
| images | TEXT | 无 | 图片URL列表（JSON数组，可空，支持多张） |
| status | VARCHAR(64) | DEFAULT 'pending' | 包裹状态 |
| estimated_delivery | DATETIME | 无 | 预计送达时间（可空） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 状态（接口层约束）

- `status` 合法值：`pending`、`received`、`in_transit`、`arrived`、`delivered`、`exception`

---

## 2.5 `orders`（订单表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 订单ID |
| user_id | INT | NOT NULL, INDEX `idx_orders_user` | 下单用户ID（逻辑关联 `users.id`） |
| parcel_id | INT | INDEX `idx_orders_parcel` | 关联包裹ID（可空，逻辑关联 `parcels.id`） |
| total_amount | DECIMAL(10,2) | NOT NULL | 订单金额 |
| currency | VARCHAR(8) | DEFAULT 'TWD' | 币种 |
| status | VARCHAR(64) | DEFAULT 'pending', INDEX `idx_orders_status` | 订单状态 |
| payment_method | VARCHAR(64) | 无 | 支付方式（可空） |
| notes | VARCHAR(255) | 无 | 备注（可空） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 状态（接口层约束）

- `status` 合法值：`pending`、`paid`、`processing`、`shipped`、`completed`、`cancelled`

---

## 2.6 `admin_audit_logs`（管理员审计日志表）

| 字段名 | 类型 | 约束/索引 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 审计日志ID |
| admin_id | INT | INDEX `idx_audit_admin_id` | 触发操作的管理员ID（可空） |
| action | VARCHAR(64) | NOT NULL, INDEX `idx_audit_action` | 操作类型 |
| target_type | VARCHAR(64) | 无 | 操作对象类型（可空） |
| target_id | INT | 无 | 操作对象ID（可空） |
| result | VARCHAR(32) | NOT NULL | 操作结果 |
| ip | VARCHAR(64) | 无 | 请求来源IP（可空） |
| detail | VARCHAR(255) | 无 | 详情说明（可空） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP, INDEX `idx_audit_created_at` | 记录时间 |

### 常见 `action` 值（来自管理接口）

- `auth.login`
- `auth.logout`
- `admin.role_check`
- `admin.create`
- `admin.update_status`
- `admin.delete`

### `result` 值

- `success`
- `failed`
- `denied`

---

## 3. 关系说明（逻辑关系）

- `parcels.user_id` → `users.id`
- `orders.user_id` → `users.id`
- `orders.parcel_id` → `parcels.id`
- `admin_audit_logs.admin_id` → `admin_users.id`

> 当前为“应用层维护关系”，数据库层未设置 `FOREIGN KEY`。

## 4. 初始化与默认数据规则

- 系统首次启动若 `admin_users` 为空，会创建一个超级管理员。
- 必须提供环境变量：
  - `DEFAULT_ADMIN_USERNAME`
  - `DEFAULT_ADMIN_PASSWORD`（至少 12 位）
  - `DEFAULT_ADMIN_EMAIL`
- 若无 `super_admin`，系统会将最早创建的管理员提升为 `super_admin`。
