# 系统布局结构说明

> 基于 `src/app/layouts/AdminLayout.tsx` 及各 Tab 组件的实际 UI 结构整理。

---

## 1. 整体布局

```
┌──────────────────────────────────────────────────────────────────────┐
│                         Layout（全屏）                               │
│ ┌──────────┬─────────────────────────────────────────────────────┐  │
│ │          │              顶部导航栏 (Header)                    │  │
│ │          ├─────────────────────────────────────────────────────┤  │
│ │  侧边栏   │                                                    │  │
│ │ (Sider)  │           主要内容区 (Content)                      │  │
│ │          │                                                    │  │
│ │          │                                                    │  │
│ │          │                                                    │  │
│ └──────────┴─────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 2. 侧边导航栏 (Sider)

- **位置**：左侧固定定位
- **组件**：`<Sider>`
- **宽度**：展开 180px / 折叠 56px
- **主题**：`dark`（深色）

### 子结构

| 区域 | 说明 |
|---|---|
| Logo | 展开时显示"CMS跨境系统"，折叠时显示"CMS" |
| 菜单列表 | 纵向菜单项，支持选中高亮（橙色 `#f58220`）。包含：总览、会员管理、订单管理、简讯资讯、包裹管理、系统管理员 |
| 语言切换 | 底部固定，`Dropdown` 组件，支持 简体中文/繁体中文/英文/印尼文 |

```
┌──────────┐
│   Logo   │
├──────────┤
│ 总览     │
│ 会员管理  │
│ 订单管理  │
│ 简讯资讯  │
│ 包裹管理  │
│ 系统管理员│
├──────────┤
│ 🌐 语言  │
└──────────┘
```

---

## 3. 顶部导航栏 (Header)

- **高度**：45px
- **组件**：`<Header>`

### 子结构

| 位置 | 元素 | 说明 |
|---|---|---|
| 左侧 | 折叠按钮 | 控制侧边栏展开/折叠 |
| 左侧 | 面包屑 | 显示当前模块名称 |
| 左侧 | 刷新按钮 | `ReloadOutlined` 图标按钮，触发当前 Tab 数据刷新 |
| 右侧 | 用户信息 | 管理员用户名 + 角色标识 + 头像，点击展开退出登录下拉菜单 |

```
┌───────────────────────────────────────────────────────────────┐
│ ☰  面包屑 > 当前模块  🔄                    用户名 / 角色  👤 │
└───────────────────────────────────────────────────────────────┘
```

---

## 4. 主要内容区 (Content)

- **外边距**：8px
- **背景**：白色圆角卡片，`borderRadius: 8`
- **布局**：`flex` 纵向，填充剩余空间

内容区根据当前选中的菜单项渲染不同的 Tab 组件（`OverviewTab`、`UsersTab`、`OrdersTab`、`SmsTab`、`ParcelsTab`、`AdminsTab`）。

### 4.1 总览页 (OverviewTab)

独立统计卡片布局，不使用以下三层结构。

### 4.2 业务 Tab 通用三层结构

所有业务 Tab（会员、订单、简讯、包裹、管理员）共享以下纵向三层布局：

```
┌─────────────────────────────────────────────────────────────┐
│                  顶层 - 操作栏                               │
│  ┌────────────────────────────────┐  ┌──────┐               │
│  │  🔍 搜索输入框（含 allowClear）  │  │ 搜索 │               │
│  └────────────────────────────────┘  └──────┘               │
├─────────────────────────────────────────────────────────────┤
│                  中层 - 数据表格                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ 表头行 1：列标题（支持排序点击）                        │    │
│  │ 表头行 2：列级搜索输入框 / 日期范围选择器               │    │
│  ├─────────────────────────────────────────────────────┤    │
│  │                    数据行                            │    │
│  │                    数据行                            │    │
│  │                    ...                              │    │
│  └─────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────┤
│                  底层 - 分页器                               │
│              ◀ 1 2 3 ... ▶  每页 50 条  跳至 __             │
└─────────────────────────────────────────────────────────────┘
```

#### 顶层 - 操作栏

| 元素 | 说明 |
|---|---|
| 搜索输入框 | 全局关键词搜索，带 `allowClear`（点击 X 自动触发重置） |
| 搜索按钮 | 触发全局搜索请求 |

- **样式**：`padding: 12px 16px`，底部 `1px solid #f0f0f0` 分割线
- **布局**：`<Space wrap>`，不可收缩（`flexShrink: 0`）

#### 中层 - 数据表格

| 特性 | 说明 |
|---|---|
| 组件 | Ant Design `<Table>` |
| 双行表头 | 通过 `columns.children` 实现。父列：标题 + 排序；子列：列级搜索输入框 |
| 固定列 | 左侧固定：序号列（含行复选框）；右侧固定：操作列 |
| 列级搜索 | 文本列：`<Input>` 回车触发；日期列：`<DatePicker.RangePicker>` |
| 排序 | 点击父表头触发，服务端排序 |
| 虚拟高度 | 通过 `ResizeObserver` 动态计算 `scroll.y`，自适应容器高度 |
| 弹性空白列 | `spacer` 列填充剩余空间 |

- **布局**：`flex: 1, minHeight: 0, overflow: hidden`

#### 底层 - 分页器

| 元素 | 说明 |
|---|---|
| 组件 | Ant Design `<Pagination>` |
| 功能 | 页码切换、每页条数切换（10/20/30/50）、快速跳转、总条数/页数显示 |

- **样式**：`padding: 6px 16px`，顶部分割线，`flexShrink: 0`，`z-index: 10`
